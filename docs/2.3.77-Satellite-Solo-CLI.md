### [Solo CLI](https://github.com/GetSoloTech/solo-cli)

> Handle: `solo`<br/>
> URL: [http://localhost:34811](http://localhost:34811) (CLI API)

Solo is an open-source, local-first AI agent that helps you automate your tasks. It's the fastest way to deploy Physical AI on your edge devices. It manages model servers (Ollama, vLLM, llama.cpp) and supports MCP tools.

#### Starting

```bash
harbor build solo
harbor up solo
```

Solo CLI in Harbor runs as a container with access to the host's Docker socket, allowing it to manage other model server containers if needed.

#### Configuration

##### Environment Variables

Following options can be set via [`harbor config`](./3.-Harbor-CLI-Reference.md#harbor-config):

```bash
# Host port for the Solo CLI API (OpenAI-compatible)
HARBOR_SOLO_HOST_PORT=34811

# Docker image for Solo CLI
HARBOR_SOLO_IMAGE=getsolotech/solo-cli

# Version/tag for Solo CLI
HARBOR_SOLO_VERSION=latest

# Persistent workspace for Solo CLI
HARBOR_SOLO_WORKSPACE=./services/solo

# Git reference for building from source
HARBOR_SOLO_GIT_REF=https://github.com/GetSoloTech/solo-cli#main
```

##### Volumes

- `./services/solo/config`: Persists Solo configuration and authentication.
- `./services/solo/data`: Persists downloaded models and other data.
- `/var/run/docker.sock`: Allows Solo to manage other Docker containers.

#### Troubleshooting

```bash
harbor logs solo
```

If Solo cannot manage other containers, ensure the Docker socket mount is working correctly and the user has permissions.
