### [Clawdbot](https://github.com/clawdbot/clawdbot)

> Handle: `clawdbot`<br/>
> URL: [http://localhost:34721](http://localhost:34721)

![Clawdbot Screenshot](./harbor-clawdbot.png)

Clawdbot is a personal AI assistant that runs locally and connects to your existing chat channels through a self-hosted gateway and Control UI.

## Starting

```bash
# Build from source (recommended for Docker installs)
harbor build clawdbot

# Set the model (required on first start)
harbor config set clawdbot.model "llama3.1:8b"

# Start Clawdbot with a backend (Ollama, llamacpp, vllm, etc.)
harbor up clawdbot ollama --open

# Open the Control UI
harbor open clawdbot
```

- Clawdbot requires an OpenAI-compatible backend. Start with one of: `ollama`, `llamacpp`, `vllm`, `tabbyapi`, `mistralrs`, `sglang`, or `lmdeploy`.
- Harbor automatically configures Clawdbot to use whichever backend is running.
- `harbor open clawdbot` uses the configured `clawdbot.open_url` which includes the gateway token, so no manual token entry is required by default.
- If `CLAWDBOT_MODEL` is unset, Clawdbot will exit on first start and ask you to set it.

### Channel Configuration

#### Discord Setup

Before adding your Discord bot to Clawdbot, you must enable required privileged intents in the Discord Developer Portal:

1. Go to [Discord Developer Portal](https://discord.com/developers/applications)
2. Select your application
3. Navigate to **Bot** settings
4. Under **Privileged Gateway Intents**, enable:
   - **Message Content Intent** (required for reading messages)
   - **Server Members Intent** (recommended for better user management)
   - **Presence Intent** (optional)
5. Save changes

Without these intents enabled, Discord will reject the connection with error code 4014.

#### Adding Channels

To configure Discord or other chat channels:

```bash
# Interactive channel configuration
harbor run clawdbot configure --section channels
```

Or add channels directly via CLI:

```bash
# Add Discord channel
harbor run clawdbot channels add --channel discord --token <bot-token>

# Add other channels (Telegram, WhatsApp, etc.)
harbor run clawdbot channels add --channel telegram --token <telegram-token>
```

The configuration wizard will guide you through:
1. Selecting a channel (Discord, Telegram, WhatsApp, etc.)
2. Entering credentials (bot token, etc.)
3. Configuring access policies

Your channel configurations persist across container restarts thanks to Harbor's config preservation system.

## Configuration

### Environment Variables

Following options can be set via [`harbor config`](./3.-Harbor-CLI-Reference.md#harbor-config):

```bash
# Control UI (Gateway) port
CLAWDBOT_HOST_PORT=34721

# Bridge port (device nodes)
CLAWDBOT_BRIDGE_HOST_PORT=34722

# Image + build source
CLAWDBOT_IMAGE="clawdbot"
CLAWDBOT_VERSION="local"
CLAWDBOT_GIT_REF="https://github.com/clawdbot/clawdbot.git#main"

# Persistent configuration + workspace
CLAWDBOT_CONFIG_DIR="~/.clawdbot"
CLAWDBOT_WORKSPACE_DIR="~/clawd"

# Gateway settings
CLAWDBOT_GATEWAY_MODE="local"
CLAWDBOT_GATEWAY_BIND="lan"
CLAWDBOT_GATEWAY_TOKEN="harbor-clawdbot-token-change-me"

# Model (required)
# Specify the model ID exactly as it appears in your backend.
# Harbor automatically prepends the provider prefix (e.g., "llamacpp/") internally.
#
# For llamacpp models with vendor prefixes, include the full path:
#   unsloth/GLM-4.7-Flash-GGUF:Q8_0
#   bartowski/Qwen2.5-Coder-7B-Instruct-GGUF:Q6_K
# For Ollama models:
#   llama3.1:8b
#   qwen3-vl:8b
#
# Note: Do NOT include the backend name (e.g., "llamacpp/") in CLAWDBOT_MODEL.
# Harbor handles provider prefixing automatically.
CLAWDBOT_MODEL=""

# Context window for the model
CLAWDBOT_CONTEXT_WINDOW=16384

# Build-time extra apt packages
CLAWDBOT_DOCKER_APT_PACKAGES=""

# Auto-approve Control UI device pairing
CLAWDBOT_AUTO_APPROVE_UI=true

# Optional Claude session values
CLAWDBOT_CLAUDE_AI_SESSION_KEY=""
CLAWDBOT_CLAUDE_WEB_SESSION_KEY=""
CLAWDBOT_CLAUDE_WEB_COOKIE=""
```

### Volumes

Clawdbot persists data in the following directories:
- `CLAWDBOT_CONFIG_DIR` - Gateway configuration, credentials, and tokens
- `CLAWDBOT_WORKSPACE_DIR` - Agent workspace, skills, and session data

### Configuration Persistence

On first start, Harbor creates the initial `clawdbot.json` configuration file. On subsequent restarts, Harbor only updates the dynamic parts (gateway tokens, backend URL, model ID) while preserving all user customizations including:
- Channel configurations (Discord, Telegram, etc.)
- Custom agent settings
- Model provider customizations
- Any other manual edits to the config file

This ensures your customizations aren't lost when switching models or backends.

### First-time Control UI pairing

Clawdbot runs inside Docker, so the Control UI connects from the Docker bridge and can require device pairing. Harbor auto-approves this by default. If you disable it, run:

```bash
harbor exec clawdbot node dist/index.js devices list --json
harbor exec clawdbot node dist/index.js devices approve <request-id>
```

## Troubleshooting

### Discord: Fatal Gateway error 4014

If you see `Fatal Gateway error: 4014` in logs or the Control UI when using Discord:

**Cause:** Discord is rejecting the connection because required privileged intents are not enabled.

**Fix:**
1. Go to [Discord Developer Portal](https://discord.com/developers/applications)
2. Select your bot application
3. Navigate to **Bot** settings
4. Under **Privileged Gateway Intents**, enable:
   - **Message Content Intent** (required)
   - **Server Members Intent** (recommended)
5. Click **Save Changes**
6. Restart Clawdbot: `harbor restart clawdbot`

The bot should now connect successfully. Check logs with `harbor logs clawdbot` to verify.

### Check Logs

```bash
harbor logs clawdbot
```

### Reset Local Data

```bash
# Stop the service
harbor down clawdbot

# Remove local data (WARNING: destroys config and sessions)
rm -rf ~/.clawdbot ~/clawd

# Restart
harbor up clawdbot --open
```
