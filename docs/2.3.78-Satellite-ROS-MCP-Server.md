### [ROS MCP Server](https://github.com/robotmcp/ros-mcp-server)

> Handle: `ros-mcp`<br/>

ROS MCP Server implements the Model Context Protocol (MCP) to bridge LLMs (like Claude or GPT) with Robot Operating System (ROS). It allows LLMs to interact with ROS nodes, topics, and services.

## Starting

```bash
harbor build ros-mcp
harbor up ros-mcp
```

The server connects to a `rosbridge_websocket` node. By default, it expects it at `host.docker.internal:9090`.

## Configuration

### Environment Variables

Following options can be set via [`harbor config`](./3.-Harbor-CLI-Reference.md#harbor-config):

```bash
# Host for the ROS bridge websocket (robot side)
HARBOR_ROS_MCP_ROSBRIDGE_HOST=host.docker.internal

# Port for the ROS bridge websocket
HARBOR_ROS_MCP_ROSBRIDGE_PORT=9090

# Persistent workspace for ROS MCP Server
HARBOR_ROS_MCP_WORKSPACE=./services/ros-mcp

# Git reference for building from source
HARBOR_ROS_MCP_GIT_REF=https://github.com/robotmcp/ros-mcp-server#main
```

## Troubleshooting

```bash
harbor logs ros-mcp
```

Ensure `rosbridge_suite` is running on your robot or ROS environment and is accessible from the container.
