### [Bifrost](https://github.com/maximhq/bifrost)

> Handle: `bifrost`<br/>
> URL: [http://localhost:34950](http://localhost:34950)

Bifrost is an OpenAI-compatible LLM gateway and proxy for routing requests across model providers. Harbor runs it as a satellite API service on the shared Harbor network.

#### Starting

```bash
harbor pull bifrost
harbor up bifrost --open
```

Bifrost listens on port `8080` inside the Harbor network. From the host, use the configured URL:

```bash
http://localhost:34950/v1
```

From another Harbor service, use:

```bash
http://bifrost:8080/v1
```

#### Configuration

##### Environment Variables

Following options can be set via [`harbor config`](./3.-Harbor-CLI-Reference.md#harbor-config):

```bash
# Host port for the Bifrost UI and OpenAI-compatible API
HARBOR_BIFROST_HOST_PORT              34950

# Container image and tag
HARBOR_BIFROST_IMAGE                  maximhq/bifrost
HARBOR_BIFROST_VERSION                latest

# Persistent workspace mounted into the Bifrost container
HARBOR_BIFROST_WORKSPACE              ./bifrost

# Internal HTTP port used by Harbor integrations
HARBOR_BIFROST_APP_PORT               8080

# Internal Harbor URL for other services
HARBOR_BIFROST_INTERNAL_URL           http://bifrost:8080

# Browser URL opened by harbor open bifrost
HARBOR_BIFROST_OPEN_URL               http://localhost:34950/workspace/logs

# Placeholder bearer token for Harbor clients that require an OpenAI API key
HARBOR_BIFROST_API_KEY                sk-bifrost

# Bifrost logging controls
HARBOR_BIFROST_LOG_LEVEL              info
HARBOR_BIFROST_LOG_STYLE              json

# Provider bootstrap helper image and tag used by Harbor integration overlays
HARBOR_BIFROST_BOOTSTRAP_IMAGE        curlimages/curl
HARBOR_BIFROST_BOOTSTRAP_VERSION      8.12.1

# Ollama provider model allowlist for Harbor integration overlays
HARBOR_BIFROST_OLLAMA_MODELS          *

# llama.cpp provider registration for Harbor integration overlays
HARBOR_BIFROST_LLAMACPP_BASE_URL      http://llamacpp:8080
HARBOR_BIFROST_LLAMACPP_API_KEY       sk-llamacpp
HARBOR_BIFROST_LLAMACPP_MODELS        *

# Docker Model Runner provider registration for Harbor integration overlays
HARBOR_BIFROST_DMR_BASE_URL           http://dmr:8080
HARBOR_BIFROST_DMR_API_KEY            sk-dmr
HARBOR_BIFROST_DMR_MODELS             *

# MLX provider registration for Harbor integration overlays
HARBOR_BIFROST_MLX_BASE_URL           http://mlx:8080
HARBOR_BIFROST_MLX_API_KEY            sk-mlx
HARBOR_BIFROST_MLX_MODELS             *

# oMLX provider registration for Harbor integration overlays
HARBOR_BIFROST_OMLX_BASE_URL          http://omlx:8080
HARBOR_BIFROST_OMLX_API_KEY           sk-omlx
HARBOR_BIFROST_OMLX_MODELS            *

# Additional Docker volume mounts appended by Harbor
HARBOR_BIFROST_VOLUMES
```

##### Volumes

Bifrost uses the following mounts:

```bash
${HARBOR_BIFROST_WORKSPACE}:/app/data
./services/bifrost/config:/app/config
```

`HARBOR_BIFROST_WORKSPACE` stores Bifrost runtime data. The `./services/bifrost/config` directory is reserved for service-local configuration files.

#### Troubleshooting

Check Bifrost logs:

```bash
harbor logs bifrost
```

If the host port is already in use, choose another port:

```bash
harbor config set HARBOR_BIFROST_HOST_PORT 34960
```

If another Harbor service cannot reach Bifrost, confirm both services are running together and use the internal URL:

```bash
http://bifrost:8080/v1
```
